## 1. 目標概述  
**設計一個「印表機儀表板」**，讓長官在 5–10 秒內即可掌握：

| 主要資訊 | 為何重要 | 決策點 |
|----------|----------|--------|
| 印表機即時狀態（正常/警告/錯誤） | 判斷是否需要即時介入 | 是否派人檢查 / 立刻聯絡維修 |
| 耗材剩餘量（碳粉、墨水、紙張） | 預防印刷中斷 | 是否提前採購、調整印量 |
| 近期印刷量 & 成本（每日/每週） | 控管成本、預算 | 是否需要限制印刷、優化流程 |
| 排程列印佇列 & 預估完成時間 | 了解交付時程 | 是否調整排程、加速重要文件 |
| 故障與警告歷史（近 30 天） | 趨勢分析、預防性維護 | 是否安排保養、升級設備 |
| 服務合約到期日 & 保固狀態 | 合約管理 | 是否續約、找替代方案 |

> **關鍵原則**：**資訊層級分明**、**視覺焦點明確**、**行動呼叫 (CTA) 直接**，讓長官不需解讀繁雜文字即可直接「看」出要做什麼。

## 2. 介面布局（Desktop 版）  

### 2.1 整體分區（4 大區塊）

```
+-----------------------------------------------------------+
| ① 頂部全寬「概況條」                                      |
|   ──> 狀態指示燈、耗材總覽、成本概覽、合約到期               |
+-----------------------------------------------------------+
| ② 左側「即時狀態 & 警報」   |   ③ 中央「列印佇列」    |
|   - 印表機圖示 (綠/黃/紅)   |   - 佇列表格、預估完成時間 |
|   - 紙張、碳粉、墨水條形圖 |   - 重要/普通/低優先度標示 |
+-----------------------------------------------------------+
| ④ 底部「分析報表 & 行動建議」                              |
|   - 印刷量趨勢圖、成本曲線、耗材耗盡預測、維護建議          |
+-----------------------------------------------------------+
```

### 2.2 ① 頂部全寬「概況條」  

| 元件 | 位置 | 顯示方式 | 建議顏色/圖示 |
|------|------|----------|--------------|
| **印表機狀態指示燈** | 左端 | 大圓形 LED | ✅ 綠（正常）<br>⚠️ 黃（警告）<br>❌ 紅（錯誤） |
| **耗材剩餘** | 中間左側 | 三個小條形圖 + 數值（%）| 碳粉藍、墨水橙、紙張灰 |
| **每日成本** | 中間右側 | 數字 + 小圖表（小折線）| 文字顏色根據成本變化：↑ 紅、↓ 綠 |
| **合約到期** | 右端 | 倒數天數 + 警示圖示 | 30 天內黃警示、7 天內紅緊急 |

> **互動**：滑鼠懸停任一項目會彈出「詳細說明」視窗（如耗材預估剩餘天數、成本計算方式）。

### 2.3 ② 左側「即時狀態 & 警報」  

| 元件 | 顯示內容 | 建議設計 |
|------|----------|----------|
| **印表機圖示** | 3D 立體圖 + 狀態光環 | 光環顏色對應概況條的 LED |
| **紙張/碳粉/墨水條形圖** | 每種耗材的當前% | 用「進度條」+ 百分比文字 |
| **最近 5 筆警報** | 文字 + 時間戳 | 銀灰底，警報等級顏色（黃/紅） |
| **快速行動按鈕** | 「重新啟動」<br>「聯絡維修」<br>「查看耗材訂購」 | 按鈕採用主色調（藍）+ 圓角，右側固定浮動 |

### 2.4 ③ 中央「列印佇列」  

| 欄位 | 說明 | 建議格式 |
|------|------|----------|
| **文件名稱** | 完整檔名 | 文字截斷+tooltip完整名 |
| **提交者** | 使用者姓名/部門 | 小圖示 (部門圖標) |
| **頁數** | 數字 | 右對齊 |
| **優先等級** | 高 / 中 / 低 | 顏色標籤：紅、黃、綠 |
| **預估完成時間** | HH:MM | 進度條 + 剩餘時間 |
| **狀態** | 等待 / 列印中 / 完成 | 小圖示 (時鐘 / 打印機 / 勾) |

> **功能**：  
> • **排序**：點擊欄位標題排序（例如依「預估完成時間」升序）  
> • **篩選**：快速切換「只顯示高優先」  
> • **批次操作**：勾選多筆後可「暫停/取消」或「提升優先」  

### 2.5 ④ 底部「分析報表 & 行動建議」  

1. **印刷量趨勢圖**（折線）  
   - X 軸：日期（最近 30 天）  
   - Y 軸：印刷頁數  
   - 交叉點：高峰期自動標註（紅點）  

2. **成本曲線**（雙軸）  
   - 左 Y 軸：每頁成本（NT$）  
   - 右 Y 軸：總成本（NT$）  

3. **耗材耗盡預測**（條形）  
   - 依當前使用速率預估「剩餘天數」  
   - 超過 7 天顯示黃色警示，<3 天顯示紅色  

4. **維護建議卡**（卡片式）  
   - **預防性保養**：如「碳粉更換建議於 2026/05/10 前」  
   - **合約提醒**：如「保固到期剩 14 天」  
   - **行動按鈕**：直接跳到「建立採購單」或「排程保養」  

> **可導出**：PDF/Excel 報表（供會計或採購部門使用）

## 3. 視覺設計要點  

| 原則 | 實作方式 |
|------|----------|
| **資訊層級** | 1️⃣ 重要訊息（狀態、耗材）使用大字、鮮明顏色；2️⃣ 次要訊息（趨勢圖）使用較淡的灰階或次要色。 |
| **色彩語意** | 綠 = 正常、藍 = 常規、黃 = 警告、紅 = 緊急。避免過度使用紅色，以免造成焦慮。 |
| **圖示統一** | 采用 Material Design 或 Font Awesome 系列圖示，確保每個功能都有直觀圖示。 |
| **字型** | 主字體：Noto Sans TC（支援繁體中文），字重 400/600，標題 18‑20pt，內文 14‑16pt。 |
| **留白** | 每個卡片、欄位之間保留 12‑16px 的間距，讓資訊不會擁擠。 |
| **響應式** | 針對平板或 4K 大螢幕自動調整欄位寬度，確保在會議室投影時仍保持可讀性。 |

## 4. 行動建議（Decision‑Support）  

| 資訊 | 建議觸發條件 | 推薦行動 |
|------|--------------|----------|
| **碳粉剩餘 < 15%** | 每日消耗速率 > 500 頁 | 「自動產生採購單」+「發送 Email 給採購」 |
| **列印佇列超過 20 項** | 平均等待時間 > 10 分鐘 | 「提示主管是否提升優先級」或「暫停低優先任務」 |
| **成本上升 > 10%（較上月）** | 成本曲線斜率變陡 | 「建議檢查耗材品質或換供應商」 |
| **故障頻率 > 3 次/週** | 同一錯誤代碼重複 | 「建議更換機型或升級韌體」 |
| **合約到期 < 30 天** | 合約到期日倒數 | 「自動彈出續約表單」 |

> **實作方式**：在後端設定「規則引擎」或「簡易 AI」(例如使用 Python + Pandas 進行閾值檢測)，前端以 **Toast Notification** 彈出，並附上「立即執行」的 CTA 按鈕。

## 5. 互動流程示例（從長官視角）  

1. **開啟儀表板** → 頂部概況條呈現 **綠色 LED**，碳粉剩 18%（黃色警示），成本較上月下降 5%（綠字）。  
2. **點擊碳粉條** → 彈出「預估剩餘天數：12 天」+ 「一鍵產生採購單」按鈕。  
3. **查看列印佇列** → 發現 3 份「高」優先文件的預估完成時間已超過 30 分鐘。  
4. **立即點擊「提升優先」** → 系統重新排序，預估完成時間縮短至 5 分鐘。  
5. **滑到底部分析報表** → 趨勢圖顯示本月印量較上月上升 20%，成本上升 2%。  
6. **點擊「維護建議卡」** → 彈出保養排程表單，預設 2026/05/10 進行碳粉更換，點擊「加入行事曆」完成排程。  

> 整個過程不超過 2 分鐘，長官即可完成 **監控 + 決策**。

## 6. 技術實作建議  

| 層級 | 技術選型 | 重點說明 |
|------|----------|----------|
| **前端** | React + TypeScript + Ant Design (或 Material‑UI) | 內建圖表、卡片、通知系統，支援主題化。 |
| **圖表** | Recharts / ApexCharts / ECharts | 支援折線、條形、雙軸圖，易於自訂顏色。 |
| **狀態即時更新** | WebSocket (Socket.io) 或 Server‑Sent Events | 印表機狀態、佇列即時推送。 |
| **後端** | Node.js (Express) + PostgreSQL + Prisma ORM | 數據儲存、規則引擎、報表產出。 |
| **規則引擎** | node‑cron + custom JS 判斷 | 每 5 分鐘檢查閾值，觸發通知。 |
| **報表匯出** | jsPDF + SheetJS | 一鍵產生 PDF/Excel。 |
| **身分驗證** | OAuth2 + JWT | 確保只有授權人員能觀看/執行行動。 |
| **部署** | Docker + Kubernetes (可擴展) | 支援多家分公司同時監控多台印表機。 |

## 8. 下一步建議  

1. **確認需求細節**（如是否需要多語系、是否要支援行動端）。  
2. **安排 UI/UX 工作坊**，讓長官直接參與 Wireframe 討論，確保「一目了然」的視覺焦點符合他們的習慣。  
3. **建立原型**（Figma）並邀請 2‑3 位主管進行 15 分鐘的可用性測試，收集「最想先看到」的資訊優先級。  
4. **迭代開發**：先交付 MVP（概況條 + 列印佇列），再逐步加入分析報表與自動化決策支援。
